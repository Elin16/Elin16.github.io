5.2 时间效率

一些影响时间效率的细节： - C/C++中的引用传值vs值转递，后者从形参到实参会产生一次拷贝操作。 - 循环vs递归，递归更简洁，但时间效率可能更差。实践中，可以用递归思考问题，用数组来保存中间结果，基于循环实现。 - 算法，根据输入数据的特点采用最合适的算法。 - 面试时，可以在第一时间告诉面试官最直观的算法，然后一步步优化，逼近最优算法，展现态度和激情。

P25：数组中出现次数超过一半的数字

要点： - 时间复杂度的考虑：排序后搜索vs统计数字出现次数 - 全面性：给定的输入非法，如空指针、数组长度为1、不存在次数超过一半的数字 - 应该写代码检查，并规定非法情况的返回值 - 得到数组中任意第k大的数字的通用算法 - 基于快速排序的partition函数实现，当选中下标为k时，该数即为第k大的数 - O(n)

P42：连续子数组和的最大值

我有解法3: 采用前缀和分析连续子数组和

思路：如果给出数组的前缀和数组prefixSum，则连续子数组[i, j)的和为prefixSum[j] = prefixSum[i]。要求连续子数组和的最大值，则是在保证i < j的情况下，最小化prefixSum[i]，最大化prefixSum[j]。

实现：O(n)扫描数组，维护变量minPrefixSum、currentSum、maxSubArraySum

int FindGreatestSumOfSubArray(vector<int>& nums) {
  if (nums.length == 0)
    return 0;
  int minPrefixSum = 0, currentSum = 0, maxSubArraySum = 0x80000000; // in case all inputs are minus
  for (vector<int>::iterator i = nums.begin(); i != nums.end(); ++i) {
    currentSum += *i;
    if (currentSum < minPrefixSum) 
      minPrefixSum = currentSum;
    if (currentSum - minPrefixSum > maxSubArraySum)
      maxSubArraySum = currentSum - minPrefixSum;
  }
  return maxSubArraySum;
}

P43： 1~n整数中1出现的次数

  • 题目：给定n，求1~n整数中1出现的次数
  • 思路：
  • 暴力循环，O(nlogn)
  • 递归，每次减少1位最高位，O(logn)
  • 心得：
  • 在问题高度重复的情况下，要善用找规律和递归分析问题
  • 本题的巧妙之处在于，问题的解只和数位上的数字有关，和数字大小无关，由此以数位为拆解单位，用数位来划分递归层次，达到了化简问题的目的
